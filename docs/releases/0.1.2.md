# Fugue 0.1.2

Fugue 0.1.2 turns the controlled experiment operator into a governed
laboratory that an external research loop can use without receiving execution
authority, credentials, a writable application checkout, or a Docker socket.

The release keeps Fugue's original execution path:

```text
ResolvedRunPlan → PlannedCell → RunSnapshotV1 → PredictionRowV1
```

It adds a durable research path around that operator:

```text
Research → controlled Study → admitted Run → evaluation → sourced Result
```

The outer loop remains the researcher. Fugue owns locking, policy, approval,
execution, evidence reconciliation, evaluation, and accounting.

## What is included

### Governed campaigns and authored tasks

- Campaign policy bounds stages, variants, tasks, attempts, concurrency, calls,
  and cost before execution.
- Outer loops may draft typed tasks, scenarios, resources, interactions,
  criteria, and judges. Drafts cannot carry raw host paths, credentials, setup
  commands, arbitrary dependencies, or mutable source references.
- Preview is pure. Starting an experiment locks an accepted task suite and
  compiles it through the existing Fugue operator; there is no second executor.
- Discovery tasks may adapt to earlier evidence. Confirmatory holdouts must have
  been locked before the outcome used to choose an intervention.
- Deterministic benchmark outcomes remain distinct from authored criteria and
  judge results. Existing prediction rows can be rescored without rerunning an
  Agent.

### Durable Research and Study records

- A Research record keeps the question, brief, append-only notes, Study lineage,
  selected evidence, open questions, and immutable Results.
- Controlled-Study parentage forms a DAG. Every preview, task suite, plan, outcome,
  and analysis remains content-addressed.
- Results state their scope, estimate or outcome, uncertainty, exclusions, and
  exact evidence references. Corrections add a superseding Result rather than
  mutating history.
- Operation IDs and optimistic Research revisions make retries explicit and
  idempotent.
- A SQLite WAL store and leased queue survive process restarts. Already-launched
  trials are reconciled through the campaign ledger and are never silently
  launched twice.

### External-Agent interface

- `fugue[research]` exposes the same service through Python, typed REST,
  resumable SSE, and Streamable HTTP MCP.
- `fugue research serve` is the bearer-authenticated control plane.
- `fugue research worker` performs preparation and execution without a public
  port.
- The Compose deployment gives the Docker socket only to the worker. The
  control service has a read-only root filesystem, dropped capabilities, and no
  execution approval tool.
- `fugue research approve` is a separate operator action bound to one exact
  preview digest, maximum cell count, and spend cap.
- The portable `optimize-agent-with-fugue` skill teaches evidence review,
  controlled task construction, preview, approval handoff, and bounded
  interpretation. The packaged MCP prompt is generated from the same source.

### Trace-grounded research loops

- Registered Weave and normalized JSONL sources provide bounded, redacted trace
  audits through immutable references.
- Trace content is treated as untrusted data. The Agent receives normalized
  observations and safe samples, never hidden credentials or an instruction
  channel from a trace.
- Reviewed failures can become authored discovery tasks and separately locked
  holdouts.
- One research cycle may reconcile a terminal parent, record a sourced Result,
  and preview one lineage-bound child experiment. It always stops before
  approving or starting the child.

### Generic research-record projection

- `ResearchLogEventV1` publishes lifecycle, evidence, budget, limitation, and
  Result events through an outbox.
- JSONL and authenticated HTTP sinks use producer event IDs for idempotency and
  reject conflicting replays.
- Sink failure never changes a Run outcome or launches work again.
- Publication contains aggregates and immutable references only. It excludes
  prompts, trace bodies, credentials, private criteria, expected values, gold
  paths, and hidden reasoning.
- Consumers can project Fugue Research and Studies beside other research systems without
  turning the console into an executor or a second evidence store.

## Qualified example

The combined behavior underlying 0.1.2 was qualified at source commit
`d323d4854025343c76451cd44fa6491121f8b53a` with a trace-grounded support-data
authority study. The later release commit changes version metadata,
documentation, its expected-version fixture, and CI coverage; it does not
rerun or relabel the paid study as exact-head evidence.

```text
1 paired synthetic task × 3 loop designs × 2 harnesses × 1 attempt = 6 cells
```

The experiment held GLM-5.2, the task, runtime, and attempt fixed. It compared
current behavior, a safety reminder, and an explicit check of risky actions
against the user's request across Codex and Claude Code.

- all six cells reached terminal, reconciled evidence;
- prediction, conversation, trace, and root identities were unique;
- route receipts, Agent links, runtime locks, and accounting were complete;
- the worker recovered after one restart without a duplicate launch;
- observed cost was `$1.5325047455785767`;
- the explicit risky-action check passed in both harnesses; the other four
  cells did not pass.

This is one synthetic task and one attempt. It demonstrates a trustworthy
research flow, not a general safety claim, harness ranking, or replicated
capability estimate. The terminal evidence does not expose a reliable failure
cause for the four failed cells, so the release does not invent one.

## Security and interpretation boundaries

- Agent-facing credentials can create drafts, notes, trace audits, and pure
  previews. They cannot approve spend.
- Active Harbor trials cannot install, download, build, start services, access
  arbitrary host paths, or reach Docker.
- Candidate repositories and artifacts must come from registered immutable
  sources. The external Agent changes code elsewhere and submits an immutable
  reference.
- Required evaluator failure makes evaluation unavailable; it does not become
  an Agent task failure.
- Missing usage remains unavailable instead of zero. Admission reserves are not
  reported as observed cost.
- Meta-analysis reports aligned task and scenario effects, uncertainty, judge
  disagreement, behavior, latency, tokens, and cost without emitting a
  universal harness ranking.

## Upgrade notes

- The package version is `0.1.2`; Python 3.12 and 3.13 remain supported for the
  core operator, while Harbor-backed research workers require Python 3.13.
- The research extras are optional: install `fugue[research]` for the control
  API and `fugue[research-worker]` for the worker dependency closure.
- Existing operator experiments continue to use the same canonical planning,
  execution, and result contracts.
- Research records, Studies, and research events are append-only. Do not rewrite an old
  Result to correct it; add a new Result with `supersedes`.
- Visualization is optional. An unavailable research-record sink may leave
  publication pending, but it cannot change a completed Run.
