# Fugue 0.1.1

Fugue 0.1.1 is a hardening release stacked on the frozen 0.1 draft at
`eec452c19a3eff47e88441c1ef311a04dd43143c`. It corrects evidence units,
finishes the managed memory-system boundary, and repairs Codex native MCP
without changing the 0.1 source pull requests.

## Release contract

- `RunSnapshotV3` records resolved capability decisions, logical predictions,
  runtime locks, publication schema, cohort and treatment-selection identity,
  the host evaluation-lock digest, and both source and resolved experiment
  digests before execution. V1 and V2 snapshots remain readable.
- `PredictionRowV2` is the single result consumed by export, analysis, run
  summaries, reports, and W&B publication. Retrieval and episode measurements
  remain separate raw evidence.
- Publication is idempotent by project, prediction ID, scorer version, and
  revision. Explicit republishing records a reason and a `supersedes` link.
- Context delivery is required in authored experiments. Applicability is a
  typed decision over workload, system, delivery, harness, and model route.
- Codex uses an isolated per-cell `CODEX_HOME`. W&B carries model traffic;
  Fugue's gateway carries allowlisted native MCP traffic. The two routes are
  never conflated or converted.
- A treatment cell must prove required skill and context registration before
  execution. Invocation evidence is recorded separately from registration.
- Direct diagnostics remain ordinary Weave operations and never synthesize an
  Agent, conversation, or `invoke_agent` root.
- Gold localization data remains in a private host-only evaluation lock. Raw
  paths never enter a container, rendered job, snapshot, Agent input, or trace;
  publication contains only derived localization metrics and the lock digest.

## Managed memory systems

All research adapters remain experimental and outside default presets. Setup
builds or downloads the pinned runtime, workload data, context indexes, harness
images, and task images; preview and run do none of those operations.

| System | Pinned boundary | 0.1.1 behavior |
| --- | --- | --- |
| GitNexus | `1.6.3+fugue-vector.6` | Native MCP; offline BM25 or fail-closed hybrid vector; explicit PolyForm noncommercial approval gate |
| CodeGraph | 0.9.0 | Native MCP; per-platform archive digest |
| Semble | 0.5.1 | Native MCP; exact wheel, model revision, parser bundle, offline trials |
| Project RAG | `d5abf98…` | Native MCP; pinned source tree and model, isolated LanceDB state |
| Graphiti | 0.29.2 and Neo4j 5.26.12 | Managed start/status/stop lifecycle with persistent data |
| OpenWiki | 0.1.1 | Portable files from an isolated builder workspace through the Fugue bridge |
| lat.md | 0.11.0 | Native MCP; missing API key is expected `not_applicable` |

The task snapshot is read-only at `/workspace/repository`. Adapter indexes,
model caches, databases, and continuity state are isolated from it. Fugue's
gateway preserves the upstream tool schema, cancellation, and logs while
adding canonical cell correlation. Active trials cannot install packages,
download models, start services, or access the Docker socket.

GitNexus hybrid vector pins Transformers.js 4.2.0, LadybugDB 0.15.4,
onnxruntime-node 1.24.3's CPU bindings, and Snowflake Arctic Embed XS at
`d8c8652…`. Setup verifies every model and native-library digest, creates
nonzero 384-dimensional embeddings offline, and executes a lexical-mismatch
semantic probe. Vector failure is fatal for the hybrid candidate; the explicit
BM25 candidate remains legal without embeddings. Their identities and caches
are distinct.

Codex is prepared as 0.143.0 with weave-codex 0.1.1. Its release gate is a
final-head live sequence covering no context, portable context, native MCP
BM25, native MCP vector, and a reviewed skill, followed by one difficult
SWE-bench task. Unit tests and earlier-head runs cannot substitute for those
links, registrations, tool calls, or vector telemetry.

The difficult SWE task has two distinct evidence units. The ordinary
GitNexus ablation measures whether an agent chooses to use retrieval without
being prompted. The separate `gitnexus-swe-contract` canary requires one
semantic lookup before editing and validates the end-to-end transport contract.
Contract-canary rows are never merged into efficacy cohorts.

Graphiti accepts explicit compatible Neo4j credentials. Otherwise
`--start-services` generates an ignored mode-0600 credential file. Stop keeps
the named data volume; destructive purge is intentionally absent.

## Hard-memory cohorts

SWE-bench Verified is pinned at
`c104f840cc67f8b6eec6f759ebc8b2693d585d4a`. A deterministic lock excludes
tasks exposed by prior live runs, enforces the hard-memory eligibility rules,
and separates eight discovery, four untouched holdout, and four disjoint
GitNexus-ablation tasks. Raw gold paths are materialized only into the ignored
host evaluation lock.

The executable presets separate 48 context-contract cells, 32 no-context
calibration cells, 80 Latin-square discovery cells, 192 holdout cells, 96 easy
controls, 128 hard repository-QA cells, a 96-prediction GitNexus BM25/vector
ablation, and 72 PDF-skill cells: 744 primary Agent cells in total. Discovery
produces a versioned `TreatmentSelectionLockV1`; holdout, controls, and
repository-QA require that lock and reject manually selected variants.

The 16-cell prompt-orientation uptake diagnostic uses previously exposed
qualification tasks and remains outside primary efficacy denominators.

Direct diagnostics separately plan 900 general retrieval measurements, 450
GitNexus retrieval measurements, and 108 continuity sequence attempts. These
remain outside Agent semantics and efficacy denominators. Official SWE
resolution is primary; localization, invocation, latency, usage, cost, vector
contribution, and errors by origin are secondary.

SWE-bench grading uses a pinned offline verifier profile prepared in an
isolated Python environment inside each task image. The repository test
environment remains unchanged. Each locked image must demonstrate base
failure and gold-patch success with networking disabled before it can enter a
paid cohort.

## Readiness gates

The draft remains blocked on any missing planned coordinate, identity mismatch,
failed required registration, duplicate execution, ambiguous Agent link,
snapshot drift, runtime download during a trial, or unaccounted publication
revision. Release validation includes both Python versions, every managed
adapter on Linux amd64 and local arm64, Graphiti lifecycle, all four harnesses,
the PDF skill canary and study, generated evaluations, serving protocols, and
one normalized W&B export.

Qualification freezes the code head and every runtime/model lock before live
work. Any affected code or lock change invalidates that evidence. Every
applicable final-head Agent prediction must have exactly one remotely verified
conversation deep link, and GitNexus-vector rows must prove successful vector
execution without fallback.

Credentialed live results belong in the draft pull-request report, not in Git.
The external environment file, raw logs, generated jobs, runtime state, local
reports, and Neo4j data must remain untracked.
