# Fugue 0.1.1

Fugue 0.1.1 is a hardening release stacked on the frozen 0.1 draft at
`eec452c19a3eff47e88441c1ef311a04dd43143c`. It corrects evidence units,
finishes the managed memory-system boundary, and repairs Codex native MCP
without changing the 0.1 source pull requests.

## Release contract

- `RunSnapshotV2` records resolved capability decisions, logical predictions,
  runtime locks, publication schema, and both source and resolved experiment
  digests before execution. V1 snapshots remain readable.
- `PredictionRowV2` is the single result consumed by export, analysis, run
  summaries, reports, and W&B publication. Retrieval and episode measurements
  remain separate raw evidence.
- Publication is idempotent by project, prediction ID, scorer version, and
  revision. Explicit republishing records a reason and a `supersedes` link.
- Context delivery is required in authored experiments. Applicability is a
  typed decision over workload, system, delivery, harness, and model route.
- Codex uses an isolated per-cell `CODEX_HOME`. W&B carries model traffic;
  Fugue's gateway carries allowlisted native MCP traffic. The two routes are
  never conflated or converted.
- A treatment cell must prove required skill and context registration before
  execution. Invocation evidence is recorded separately from registration.
- Direct diagnostics remain ordinary Weave operations and never synthesize an
  Agent, conversation, or `invoke_agent` root.

## Managed memory systems

All research adapters remain experimental and outside default presets. Setup
builds or downloads the pinned runtime; preview and run do neither.

| System | Pinned boundary | 0.1.1 behavior |
| --- | --- | --- |
| GitNexus | 1.6.3 | Native MCP; explicit PolyForm noncommercial approval gate |
| CodeGraph | 0.9.0 | Native MCP; per-platform archive digest |
| Semble | 0.5.1 | Native MCP; exact wheel, model revision, parser bundle, offline trials |
| Project RAG | `d5abf98…` | Native MCP; pinned source tree and model, isolated LanceDB state |
| Graphiti | 0.29.2 and Neo4j 5.26.12 | Managed start/status/stop lifecycle with persistent data |
| OpenWiki | 0.1.1 | Portable files from an isolated builder workspace through the Fugue bridge |
| lat.md | 0.11.0 | Native MCP; missing API key is expected `not_applicable` |

The task snapshot is read-only at `/workspace/repository`. Adapter indexes,
model caches, databases, and continuity state are isolated from it. Fugue's
gateway preserves the upstream tool schema, cancellation, and logs while
adding canonical cell correlation. Active trials cannot install packages,
download models, start services, or access the Docker socket.

Graphiti accepts explicit compatible Neo4j credentials. Otherwise
`--start-services` generates an ignored mode-0600 credential file. Stop keeps
the named data volume; destructive purge is intentionally absent.

## Readiness gates

The draft remains blocked on any missing planned coordinate, identity mismatch,
failed required registration, duplicate execution, ambiguous Agent link,
snapshot drift, runtime download during a trial, or unaccounted publication
revision. Release validation includes both Python versions, every managed
adapter on Linux amd64 and local arm64, Graphiti lifecycle, all four harnesses,
the PDF skill canary and study, generated evaluations, serving protocols, and
one normalized W&B export.

Credentialed live results belong in the draft pull-request report, not in Git.
The external environment file, raw logs, generated jobs, runtime state, local
reports, and Neo4j data must remain untracked.

## Deliberately deferred

lat.md semantic execution is not a release gate without its API key. Registry
push, deployment management, autoscaling, stored conversations, media,
server-side tools, compaction, WebSockets, destructive service cleanup, and
automated curator execution remain out of scope.
