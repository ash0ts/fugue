# Fugue 0.1

Fugue 0.1 establishes one reproducible path from experiment definition to a
candidate result and, when eligible, a packaged text service.

## Release contract

- One resolved candidate model feeds Harbor rendering, run snapshots, results,
  presets, packaging, and serving.
- Candidate identity contains behavioral configuration only. Operational policy
  lives in a separate execution fingerprint.
- The operator writes a secret-free, digest-verified input lock before changing
  a run to `running` or executing a cell.
- Experiment YAML is strict: `skills`, declared context `delivery`, additive
  typed integrations, and typed scorer selections.
- Run inspection groups cells by candidate and reports completeness and an
  exact packageability reason.
- Generated evaluations are explicit, deterministic proposals with exact suite,
  workload, size, source, and judge requirements.
- Candidate packaging is limited to clean tracked source and supported
  MCP-free serving deliveries. Integrations are not packageable in 0.1.
- The optional Python 3.13 gateway provides bounded, stateless, text-only Open
  Responses, Chat Completions, and AG-UI compatibility.
- Manual and live curator workflows import one policy and cannot modify code,
  tests, workflows, dependencies, datasets, presets, README, or vendored skills.

## Deliberately deferred

Fugue 0.1 does not include integration-backed serving, registry push,
deployment management, autoscaling, stored conversations, server-side tools,
media, compaction, WebSockets, promoted self-evaluation presets, or default
research adapters. OpenWiki remains explicitly selectable for isolated review,
but is not in the memory-study presets until its host shell boundary is
independently verified.

## Manual release gates

The release owner must separately authorize and run:

- credentialed evaluation generation;
- a real-provider serving smoke;
- the live curator workflow;
- the portable/native eight-cell canary; and
- a clean Docker package, start, readiness, and protocol smoke.

These gates can incur external calls or create GitHub state and are therefore
not run automatically during local consolidation.
